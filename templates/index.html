This tool showcases how different approaches to differential privacy might affect top-viewed lists. In reality, this approach would likely be applied to e.g., pageviews by country, but here we use the top-viewed articles in a wiki (public data) as a proxy.

Top-viewed Articles on a Wikipedia

This tool fetches the top-viewed articles for a given wiki from yesterday. On the left is the actual data -- i.e. accurate counts without any noise added. On the right is the data after differential privacy has been applied (specifically noise drawn from a Laplace distribution).

You can play around with the different hyperparameters to see how it affects the results. See this Facebook blogpost for a good worked example.

Language code: which Wikipedia language to query -- e.g., en for English Wikipedia.

Epsilon (Îµ): privacy parameter. Default to 1 but the smaller you make it, the more privacy-preserving.

Sensitivity: the maximum amount that any individual can add to the result. For example, if this is daily user pageviews, 800 would be appropriate as IP+UA hashes that are associated with more than 800 pageviews in a day are labeled as "automated". If this is number of unique devices that viewed an article, it could be set to 1 as a given UA+IP could only contribute at most 1 to the count.

Min-count: not formally part of differential privacy, but we introduce a minimum number of data points -- i.e. minimum number of pageviews -- to show up on the list. 1 keeps all results; 100 or any number in that magnitude is likely a good choice.
